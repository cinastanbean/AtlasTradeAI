# MVP 功能边界与优先级清单

## 1. 文档目的

本文档用于明确 AtlasTradeAI 第一阶段 MVP 的建设边界，避免项目一开始做得过大、过散。

## 2. MVP 目标

第一阶段 MVP 的核心目标不是做全功能经营系统，而是完成一个最小可行闭环：

统一订单主视图 + 关键事件触发 + 任务中心 + 异常中心 + 基础跟单员 Agent

## 3. MVP 必做范围

建议第一阶段必须完成：

- 客户主视图
- 订单主视图
- 订单状态机
- 订单里程碑
- 统一事件模型
- 任务中心
- 异常中心
- CRM / ERP 关键数据同步
- 钉钉提醒
- 跟单员 Agent 基础能力

## 4. MVP 可选范围

如资源允许，可加入：

- 基础 BI 看板
- 回款视图
- 单证状态视图

## 5. MVP 暂缓范围

第一阶段建议暂缓：

- 复杂多 Agent 协同
- 全自动审批流
- 深度 CRM 二次开发
- 复杂成本核算重构
- 全量单证自动生成
- 大规模 AI 自动执行

## 6. 优先级清单

```mermaid
flowchart LR
    A["P0<br/>订单主线与同步"] --> B["P1<br/>事件、任务、异常"]
    B --> C["P2<br/>跟单员 Agent"]
    C --> D["P3<br/>分析与扩展模块"]
```

### 6.1 P0

- 客户 / 订单主视图
- CRM / ERP 数据同步
- 统一订单 ID 映射

### 6.2 P1

- 订单状态机
- 统一事件模型
- 任务中心
- 异常中心
- 钉钉通知

### 6.3 P2

- 跟单员 Agent
- 风险摘要
- 跟进建议
- 自动创建跟进任务

### 6.4 P3

- BI 看板
- 单证 Agent
- 回款 Agent
- 经营分析 Agent

## 7. 验收标准建议

第一阶段建议以以下结果作为验收标准：

- 能看到统一订单主视图
- 能识别关键订单状态变化
- 能由关键事件自动生成任务
- 能对高风险问题形成异常记录
- 能通过钉钉提醒责任人
- 跟单员 Agent 能输出可用的跟进建议

## 8. 文档结论

MVP 的关键，不是做多，而是形成闭环。

只要第一阶段能够跑通“订单变化 -> 事件 -> 任务 / 异常 -> 跟单员 Agent -> 钉钉提醒”这一条链路，项目就已经具备了清晰的落地基础。
